The problem

Status reports that take hours shouldn't exist.

  • ✕ You spend 30–60 minutes every week manually aggregating GitHub issues into a doc
  • ✕ Stakeholders ask for updates you already have — buried in project boards
  • ✕ Context switching breaks your flow, and stale reports lose trust over time
  • ✕ Every team writes these differently — no consistency, no audit trail
😓 The old way
✕ Open GitHub project board
✕ Filter by "updated this week"
✕ Copy-paste issue titles into a doc
✕ Manually summarize each item
✕ Format for non-technical stakeholders
✕ Email / Slack the report to the team
⏱ ~45 minutes every week

How it works

From project board to boardroom in 30 seconds.

No complex setup. No integrations. Just paste a URL and get an executive report.

1
Paste your GitHub Project URL

Point PulseReport at any GitHub Projects V2 board — org or personal

2
Choose AI model & lookback window

Pick Gemini or OpenAI, set how many days back to analyse, done

3
Get your executive report instantly

A structured markdown report lands in seconds — copy, download, or share
